# Goal TUI Command Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace `/goal` chat-feedback behavior with an action-only TUI command whose chat output is always model-visible context.

**Architecture:** Keep the existing server plugin for model/runtime behavior. Add a separate `./tui` plugin entry that registers `/goal` as a TUI action, shows dialogs/toasts, mutates the same goal store, and uses `promptAsync` only for set/replace/resume model-visible context.

**Tech Stack:** TypeScript, Bun tests, OpenCode server plugin API, OpenCode TUI plugin API, `@opencode-ai/sdk` v1 and v2 types.

## Global Constraints

- Always use TDD: write a failing test, verify it fails, implement the minimal code, verify it passes.
- Do not use ignored chat text as `/goal` feedback.
- Any text appearing in the chat transcript must be model-visible.
- Short feedback must use TUI toast with `api.ui.toast({ variant, title: "goal", message })`.
- `/goal show`, `/goal pause`, `/goal resume`, and `/goal drop` must not submit an empty or ignored prompt.
- Set, replace, and resume may call `promptAsync`; the prompt text must include the active goal context generated by `renderActiveGoalContext`.
- Normal goal completion remains model-driven through `goal({ op: "complete" })`.
- Do not add token/time budgets.
- Do not publish npm. Keep GitHub tag install and avoid `scripts.build`.
- Do not commit unless the user explicitly asks for a commit.

- Registers `/goal` as an action-only slash command through `api.keymap.registerLayer`.
- `show` opens a dialog with rendered context and does not call `promptAsync`.
- `show` with no active goal emits toast `No active goal`.

- `drop` changes status to dropped, emits toast `Goal dropped`, suppresses continuation, and does not call `promptAsync`.
- `set` and `replace` write active goal state, emit success toast, and call `promptAsync` with a text part containing `<active_goal_context>` and the objective.
- `resume` changes status to active, emits toast `Goal resumed`, and calls `promptAsync` with `<active_goal_context>`.

Expected: FAIL because `src/tui.ts` does not exist.

- [ ] **Step 3: Implement TUI command**

Create `src/tui.ts` with these functions:

```ts
export function currentSessionID(api: { route?: { current?: { params?: Record<string, unknown> } } }): string | undefined {
  const value = api.route?.current?.params?.sessionID;
  return typeof value === "string" ? value : undefined;
}

export function goalStartPromptText(context: string, action: "set" | "replace" | "resume"): string {
  const instruction = action === "resume"
    ? "Resume working toward the active goal."
    : action === "replace"
      ? "Begin working toward the replacement active goal."
      : "Begin working toward the active goal.";
  return `${context}\n\n${instruction}\nIf the goal is now complete, call goal({ op: "complete" }).`;
}
```

Implement action helpers that mutate `GoalStore`, call `api.ui.toast`, call `api.ui.dialog.replace` for show and prompts, and call `api.client.session.promptAsync` for set/replace/resume. Register through `api.keymap.registerLayer({ priority: 900, commands: [...] })` with `namespace: "palette"`, `name: "goal.menu"`, `slashName: "goal"`, and fallback to legacy `api.command.register`.
